1.变量

变量分为 可变变量（mutable）和 不可变变量（immutable)
Rust的变量默认是不可改变的，如果想改变变量值，就要声明可变变量（加 mut ）

1.1 声明不可变变量

1.2 声明可变变量

2.常量：

使用const来定义一个常量，定义常量需要明确指定数据类型

注意：

  1. 必须显示指定数据类型
  2. 常量名使用大写字母
  3. 必须使用常量表达式进行赋值，即必须是编译期能计算出的值。
  4. 不支持重定义（遮蔽）

2.1 代码示例：

2.2 不可变量与常量的区别：

  1. 常量使用const声明，不可变变量使用let来声明
  2. 常量在编译期就确定值，运行时不改变。不可变变量要到运行期才能确定。
  3. 常量不可以修改，不可变变量可以通过shadowing来修改。
  4. 常量必须声明类型，不可变变量不需要声明类型。
  5. 常量必须使用大写字母，不可变变量不需要。

3.静态变量：

3.1 说明：

  1. 静态变量使用static进行声明，并在编译时分配内存，在程序运行期间一直存在。
  2. 静态变量可以在整个程序中共享，并且在多个线程之间共享。
  3. 静态变量的值在程序运行期间不会改变，因此它它们通常用于存储程序全局状态。

3.2 代码示例

3.3 静态变量与常量的相似之处：

  1. 定义方式： 静态变量使用 static,常量使用 const
  2. 名称都要求使用大写
  3. 都必须明确指定类型
  4. 两者都要求必须使用常量表达式进行赋值，即必须是编译期能计算出的值
  5. 两者都不支持重定义（遮蔽），这点和变量不同

3.4 静态变量与常量的区别：

  1. 常量在编译时被内联，但静态变量不会。在整个程序中静态变量只有一个实例，也就是说所有引用都指向同一个地址。
    const A : i32 = 1；就是地址A（0x456）的空间里面装了 1 这个值.
    static A : i32 = 1; 就是地址A（0x789）的空间里装了0x123这个地址，指向了0x123这个地址，0x123里面刚好装了 1 这个值，然后读A就读到0x123里面那个1，也就是指针指向值所在的内存地址。
  2. 常量不可变，而静态变量和普通变量一样，默认不可变，但可以通过 mut 关键字定义为可变。

4.作用域和遮蔽（variable shadowing)

变量绑定有一个作用域(scope)，它被限定只在一个代码块(block)中生存（live)。代码块是一个被 “{ }”包围的语句集合。另外也允许变量遮蔽（variable shadowing)。

3.使用Cargo创建一个项目

1.什么是Cargo

Cargo 是一个工具,允许 Rust 项目声明其各种依赖项，并确保您始终获得可重复的构建。

为了实现这一目标,Cargo 做了四件事:

  • 引入两个，包含各种项目信息的元数据文件。
  • 获取，并构建项目的依赖项.
  • 正确使用参数，以调用rustc或其他构建工具，构建你的项目。
  • 介绍，更容易使用 Rust 项目的约定(规范/风格)。
